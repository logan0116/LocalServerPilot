# LocalServerPilot PRD

## 1. 项目概述

**项目名称**: LocalServerPilot  
**项目类型**: 前后端分离的服务器管理平台  
**核心价值**: 帮助团队管理和监控本地服务器集群，支持服务快速部署、状态监控和运维操作。  
**目标用户**: 开发团队、运维人员、AI/ML 工程师

---

## 2. 需求背景

当前项目为基于 Streamlit 的单体应用，存在以下问题：
- UI 交互体验受限
- 难以与现有 CI/CD 流程集成
- 无法支撑移动端或第三方系统调用
- 前后端强耦合，扩展困难

**改造目标**: 拆分前后端，构建 RESTful API 后端服务，支持多客户端接入。

---

## 3. 功能需求

### 3.1 服务器管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 服务器列表 | 查看所有已注册服务器基本信息（名称、IP、用户名） | P0 |
| 服务器连接测试 | 测试与服务器的 SSH 连接是否可达 | P0 |
| 新增服务器 | 添加新服务器到管理池 | P1 |
| 编辑服务器 | 修改服务器信息 | P1 |
| 删除服务器 | 从管理池移除服务器 | P1 |

### 3.2 状态监控

| 功能 | 描述 | 优先级 |
|------|------|--------|
| GPU 状态 | 实时查看服务器 GPU 利用率、显存、温度 | P0 |
| 容器状态 | 查看服务器上运行的 Docker 容器列表及状态 | P0 |
| 状态轮询 | 定时刷新服务器状态数据 | P0 |
| 状态推送 | 支持 WebSocket 实时推送状态更新 | P1 |

### 3.3 配置卡片管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 配置列表 | 查看所有服务配置卡片 | P0 |
| 配置详情 | 查看单个配置卡片的完整信息 | P0 |
| 新增配置 | 创建新的服务配置卡片 | P1 |
| 编辑配置 | 修改现有配置卡片 | P1 |
| 删除配置 | 删除配置卡片 | P1 |

### 3.4 服务操作

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 启动服务 | 在目标服务器上执行启动命令 | P0 |
| 停止服务 | 在目标服务器上执行停止命令 | P0 |
| 查看服务状态 | 通过进程检查服务运行状态 | P0 |
| 批量操作 | 同时对多个服务器执行同一操作 | P2 |

### 3.5 用户权限（未来扩展）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户认证 | API Token 或 OAuth2 认证 | P2 |
| 权限控制 | 不同用户组对服务器/服务的访问控制 | P2 |

---

## 4. 非功能需求

### 4.1 性能
- API 响应时间 < 500ms（不含 SSH 命令执行时间）
- 支持至少 10 台服务器的并发监控
- 状态轮询间隔可配置，默认 15 秒

### 4.2 可用性
- 后端服务支持健康检查接口
- SSH 连接异常时返回明确错误信息，不阻塞其他请求

### 4.3 安全性
- SSH 密码不铭文存储（可配置环境变量或密钥管理服务）
- API 请求可选认证（初期可匿名，后续扩展）

### 4.4 可维护性
- 配置信息外置（JSON/YAML/环境变量）
- 清晰的日志记录
- 模块化设计，便于功能扩展

---

## 5. 前端需求（独立项目）

前端建议使用 Vue3 + Element Plus 或 React + Ant Design，具体需求不在本文档范围。

关键前端页面：
1. **Dashboard** - 服务器状态总览
2. **服务器管理** - 服务器增删改查
3. **配置卡片** - 配置管理
4. **服务控制台** - 服务启停操作

---

## 6. 数据持久化

- **数据库**: SQLite（轻量级，无需额外依赖）
- **数据表**: 服务器信息 (servers)、服务配置 (service_configs)
- **迁移**: 支持从 JSON 文件导入现有数据

## 7. 不在本次范围

- 用户认证系统
- 多用户支持
- 移动端适配
- 服务日志收集
- 告警通知

---

## 8. 成功标准

1. FastAPI 后端提供完整的 RESTful API
2. 前端可独立部署，通过 API 与后端通信
3. 所有原有 Streamlit 功能均可通过 API 实现
4. 支持 WebSocket 实时状态推送
5. 项目文档完整（PRD + TP + API 文档）
